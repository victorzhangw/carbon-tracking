## 修改原則

### 1. 比例縮小原則

- 服務人數相關: 縮小至 6.67%（80/1200）
- 成本相關: 按實際比例調整
- 滿意度: 略為保守（降低 3-5%）
- 效率指標: 略為保守

### 2. 數據一致性

- 確保各報告間數據一致
- 確保計算邏輯正確
- 確保前後數據可對應

### 3. 用語調整

- 「正式營運」→「試營運」
- 「大規模推廣」→「小規模驗證」
- 「顯著成效」→「初步成效」
- 「全面導入」→「試點導入」

## 具體修改示例

### 示例 1: 執行概況

原文:

```
服務對象
- 獨居長者人數：1,200 人
- 服務區域：15 個行政區
- 服務據點：28 個社區關懷據點

執行人力
- 社工人員：85 人
- 志工協力：320 人
- 技術支援：12 人
```

修改為:

```
服務對象
- 獨居長者人數：80 人
- 服務區域：2 個行政區（1個都會區、1個偏鄉地區）
- 服務據點：3 個社區關懷據點

執行人力
- 社工人員：15 人
- 志工協力：25 人
- 技術支援：2 人
```

### 示例 2: 滿意度調查

原文:

```
調查對象: 210位長者（隨機抽樣）
有效問卷數: 189份
回收率: 90%
整體滿意度: 87.3%
```

修改為:

```
調查對象: 40位長者（隨機抽樣50%）
有效問卷數: 36份
回收率: 90%
整體滿意度: 82.5%
```

### 示例 3: 成本對照

原文:

```
導入前月平均: 794,400元
導入後月平均: 444,700元
月節省金額: 349,700元
```

修改為:

```
導入前月平均: 47,960元
導入後月平均: 25,430元
月節省金額: 22,530元
9個月淨節省: 15,000元（扣除系統投資）
```
