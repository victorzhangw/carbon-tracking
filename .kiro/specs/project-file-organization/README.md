## 🎯 目標

將專案根目錄的 150+ 個檔案重新組織到 8 個主要目錄中：

- `config/` - 配置檔案
- `docs/` - 文檔
- `scripts/` - 工具腳本
- `tests/` - 測試
- `assets/` - 靜態資源
- `data/` - 資料檔案
- `modules/` - 功能模組
- `archive/` - 歸檔檔案

## 🚀 如何開始執行

### 方式 1: 使用 Kiro 執行任務

1. 開啟 `tasks.md` 檔案
2. 點擊任務旁的 "Start task" 按鈕
3. Kiro 會自動執行該任務

### 方式 2: 手動執行

1. 閱讀 `design.md` 了解完整設計
2. 按照 `tasks.md` 的順序逐步執行
3. 每完成一個任務，標記為完成

## ⚠️ 重要提醒

- **執行前必須備份**: 任務 1 會建立完整備份
- **逐步執行**: 不要跳過任務，按順序執行
- **測試驗證**: 每個階段完成後都要測試
- **記錄變更**: 所有移動都會記錄在 migration_log.json
