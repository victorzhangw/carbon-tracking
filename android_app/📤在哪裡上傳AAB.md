# 📤 在哪裡上傳 AAB - 超詳細圖文指南

## 🎯 完整流程

### 前置準備

**確認你已經有：**

- [x] AAB 檔案已建置
- [x] Google Play 開發者帳號已註冊（$25）
- [x] 已登入 Google Play Console

## 📍 上傳 AAB 的位置

### Step 1: 進入 Google Play Console

```
1. 打開瀏覽器
2. 訪問：https://play.google.com/console
3. 使用你的 Google 帳號登入
```

### Step 2: 選擇或建立應用程式

#### 如果是第一次：

```
1. 點擊右上角 "建立應用程式" 按鈕
2. 填寫基本資訊：
   - 應用程式名稱：碳排放追蹤系統
   - 預設語言：繁體中文 (台灣)
   - 應用程式類型：應用程式
   - 免費或付費：免費
3. 勾選聲明
4. 點擊 "建立應用程式"
```

#### 如果已經建立：

```
1. 在應用程式列表中
2. 點擊你的應用程式名稱
```

### Step 3: 找到上傳 AAB 的地方

**在左側選單中：**

```
1. 點擊 "發布" 或 "Release"
   （左側選單，通常在中間位置）

2. 選擇 "正式版" 或 "Production"
   （如果是測試，選擇 "內部測試" 或 "封閉式測試"）

3. 點擊 "建立新版本" 或 "Create new release"
```

### Step 4: 上傳 AAB

**現在你會看到上傳區域：**

```
┌─────────────────────────────────────────┐
│  App bundles                            │
│                                         │
│  ┌───────────────────────────────────┐ │
│  │                                   │ │
│  │   拖曳檔案到這裡                   │ │
│  │   或                              │ │
│  │   [選擇檔案]                      │ │
│  │                                   │ │
│  └───────────────────────────────────┘ │
│                                         │
│  支援的檔案類型：.aab                  │
└─────────────────────────────────────────┘
```

**上傳方式：**

**方法 A：拖曳上傳**

```
1. 打開檔案總管
2. 找到：D:\python\Flask-AICares\android_app\app\release\app-release.aab
3. 拖曳到上傳區域
4. 等待上傳完成
```

**方法 B：點擊上傳**

```
1. 點擊 "選擇檔案" 或 "Browse files" 按鈕
2. 瀏覽到：D:\python\Flask-AICares\android_app\app\release\
3. 選擇：app-release.aab
4. 點擊 "開啟"
5. 等待上傳完成
```

### Step 5: 填寫版本資訊

**上傳完成後：**

```
1. 版本名稱（Release name）：
   1.0.0

2. 版本資訊（Release notes）：

   繁體中文 (台灣)：
   初始版本

   功能：
   - 訪視記錄管理
   - 碳排放計算
   - 統計分析
   - Excel 匯出

3. 點擊 "儲存" 或 "Save"
```

### Step 6: 檢查並提交

```
1. 檢查上傳的 AAB：
   ✓ 檔案名稱正確
   ✓ 版本號正確
   ✓ 沒有錯誤訊息

2. 向下滾動到底部

3. 點擊 "檢閱版本" 或 "Review release"

4. 確認所有資訊正確

5. 點擊 "開始推出至正式版" 或 "Start rollout to Production"

6. 確認提交
```

## 🆘 常見問題

### Q: 找不到 "發布" 選單？

**A: 可能的原因：**

```
1. 語言設定不同
   - 英文版叫 "Release"
   - 繁中版叫 "發布"

2. 還沒建立應用程式
   - 先點擊 "建立應用程式"
   - 填寫基本資訊

3. 開發者帳號還在審核
   - 等待審核通過
   - 檢查 Email 通知
```

### Q: 上傳後顯示錯誤？

**A: 常見錯誤：**

```
1. 版本號重複
   - 增加版本號（1.0.1, 1.0.2...）

2. 簽署金鑰不同
   - 確認使用相同的 keystore

3. 套件名稱不符
   - 確認 package name 正確
```

### Q: 可以上傳 APK 嗎？

**A: 不建議**

```
Google Play 現在要求使用 AAB
APK 只能用於內部測試
正式版必須使用 AAB
```
