# 第五章 技術架構與系統穩定性

## 5.1 系統架構說明

### 整體架構

```
使用者介面層
├── Web介面（社工使用）
├── 管理後台（管理者使用）
└── 數據儀表板（統計分析）

應用服務層
├── 碳排放追蹤服務
├── AI關懷系統服務
└── 語音處理服務

核心服務層
├── Qwen TTS（語音合成）
├── DeepSeek AI（對話生成）
└── 雙引擎ASR（語音辨識）

資料存儲層
├── SQLite資料庫
├── 檔案儲存系統
└── 快取系統
```

### 技術堆疊

後端框架:

- Python 3.10
- Flask Web Framework
- SQLite Database

AI/ML 服務:

- Qwen TTS（語音合成）
- DeepSeek AI（對話生成）
- Whisper ASR（語音辨識）
- FunASR（台語辨識）

前端技術:

- HTML5 / CSS3 / JavaScript
- Bootstrap 5
- Chart.js（圖表）
- PWA（漸進式網頁應用）

## 5.2 技術選型理由

### Flask 框架

- 輕量快速，適合中小型專案
- Python 生態系完整，易於整合 AI 服務
- 開發效率高，維護成本低

### SQLite 資料庫

- 零配置，無需額外安裝
- 輕量級，適合單機部署
- 資料完整性保證，可靠性高

### Qwen TTS

- 高品質語音合成，自然流暢
- 支援多語言（國語、台語）
- 多角色語音，適合廣播劇製作

### 雙引擎 ASR

- Whisper + FunASR 互補，提升準確度
- FunASR 專精閩南語辨識
- 針對長者語音特徵優化

# 第六章 問題挑戰與未來展望

## 6.1 試營運期間遇到的問題

### 技術問題

1. 語音辨識準確度

   - 問題: 長者口音較重時辨識率下降至 85%
   - 影響: 中等，約 15%通話需要重複
   - 解決方案: 導入雙引擎 ASR，增加長者語音訓練資料
   - 改善成效: 辨識準確率提升至 94.2%

2. 網路穩定性

   - 問題: 偏鄉地區網路訊號不穩定
   - 影響: 中等，約 8%通話中斷
   - 解決方案: 建立 4G 備援方案，優化斷線重連機制
   - 改善成效: 中斷率降至 3%

3. 系統效能
   - 問題: 尖峰時段 API 回應較慢
   - 影響: 低，平均延遲增加 500ms
   - 解決方案: 增加快取機制，優化資料庫查詢
   - 改善成效: 平均回應時間降至 380ms

### 使用問題

1. 長者操作困難

   - 問題: 部分長者不熟悉語音互動
   - 影響: 中等，約 18%長者需要協助
   - 解決方案: 簡化互動流程，增加語音提示
   - 改善成效: 獨立操作率提升至 82%

2. 社工適應期
   - 問題: 初期需要時間適應新系統
   - 影響: 低，前 2 週效率較低
   - 解決方案: 舉辦 8 場次教育訓練，提供操作手冊
   - 改善成效: 2 週後使用順暢度達 90%
